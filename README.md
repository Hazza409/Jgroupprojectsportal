# J Group Projects — Client & Builder Dashboard

A per-project dashboard for a high-end residential builder. Builders (J Group staff) see
every project; each client sees only their own. Modules: Original Estimate, Cost to Complete,
Progress Claims, Variation Register, Schedule, Calendar, Photo Library, and Drawings & Design.

## Stack & key decisions

- **Next.js 14 (App Router) + TypeScript**, **Postgres + Prisma**, **Tailwind**, **NextAuth** (Credentials + JWT).
- **Per-tenant isolation is enforced at the query layer**, not the UI — see [`src/lib/scope.ts`](src/lib/scope.ts).
  Every project-bound page/route/action calls `assertProjectAccess()` / `canAccessProject()` before
  touching data, including the file-download route. The UI also hides things, but the boundary is the query.
- **Money is integer cents** everywhere in the DB and across app boundaries — see [`src/lib/money.ts`](src/lib/money.ts). Never floats.
- **File storage is abstracted** ([`src/lib/storage/index.ts`](src/lib/storage/index.ts)) with a local-disk driver for dev
  ([`local.ts`](src/lib/storage/local.ts)) and an S3-compatible driver for prod ([`s3.ts`](src/lib/storage/s3.ts)).
- **Xero is one-directional (Xero → dashboard)** and stubbed as service interfaces with TODOs.
  The invoice-push (money movement) step is deliberately disabled and never auto-fires.

## Quick start

```bash
# 1. Install
npm install

# 2. Env
cp .env.example .env
# generate a real secret:
#   openssl rand -base64 32   → paste into NEXTAUTH_SECRET

# 3. Start local Postgres (Docker)
npm run db:up

# 4. Create schema + generate client
npm run prisma:migrate      # name the first migration e.g. "init"

# 5. Generate the example spreadsheets (also documents the parser format)
npm run make:samples

# 6. Seed users + a populated demo project
npm run seed

# 7. Run
npm run dev                 # http://localhost:3000
```

## Excel import

Both the **Original Estimate** and **Schedule** modules accept `.xlsx` uploads (builder only).
The parsers locate the header row by column aliases, so layouts can vary.

- Estimate columns: `Cost Code · Description · Qty · Unit · Unit Cost · Total` — parser: [`parseEstimate.ts`](src/lib/excel/parseEstimate.ts)
- Schedule columns: `Task · Start · Finish · % Complete` — parser: [`parseSchedule.ts`](src/lib/excel/parseSchedule.ts)

Canonical example files (generated by `npm run make:samples`):
[`examples/sample-estimate.xlsx`](examples/sample-estimate.xlsx) · [`examples/sample-schedule.xlsx`](examples/sample-schedule.xlsx)

## Connecting Xero (Cost to Complete)

One-directional: Xero → dashboard. Read-only scopes only.

1. Create an app at [developer.xero.com](https://developer.xero.com); add redirect URI
   `http://localhost:3000/api/xero/callback`.
2. Put `XERO_CLIENT_ID`, `XERO_CLIENT_SECRET`, `XERO_REDIRECT_URI`, and `XERO_TOKEN_KEY` in `.env`.
3. As a builder, open a project's **Cost to Complete** page → **Connect Xero** → consent.
4. Click **Sync now**. Supplier bills (ACCPAY) and spend bank-transactions are pulled and their
   line-item **account codes** are matched to project **cost codes**; unmatched codes are reported.
   Tokens are stored encrypted (AES-256-GCM) and auto-refresh. Syncs are incremental (`lastSyncedAt`).

Invoice push (approved claim → Xero invoice) is deliberately **not** enabled — see
[`invoicePush.ts`](src/lib/xero/invoicePush.ts).

## Deploy

- **Vercel** for the app. Set `DATABASE_URL`, `NEXTAUTH_URL`, `NEXTAUTH_SECRET`, `STORAGE_DRIVER=s3`
  and the `S3_*` vars in project env. Run `prisma migrate deploy` against your hosted Postgres
  (Neon/Supabase) on deploy.
- Switch storage to S3/R2 by setting `STORAGE_DRIVER=s3` and the `S3_*` vars — no code change.

## Assumptions made

- One primary project per client is common (high-end residential), so a client with exactly one
  project is routed straight into it.
- Estimate/Schedule sheets use the first worksheet; header row within the first 10 rows.
- Cost-code matching between estimate and Xero actuals is by exact `code` string.
- Progress-claim line totals are the claimed amount per period (cents); reconciliation-sheet
  attachment is modeled (`reconSheetKey`) and left as a follow-up upload like the others.
